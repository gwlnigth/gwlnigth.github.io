Spring

简介

Spring 是 Java EE 编程领域的一款轻量级的开源框架，由被称为“Spring 之父”的 Rod Johnson 于 2002 年提出并创立，它的目标就是要简化 Java 企业级应用程序的开发难度和周期。

Spring 自诞生以来备受青睐，一直被广大开发人员作为 Java 企业级应用程序开发的首选。时至今日，Spring 俨然成为了 Java EE 代名词，成为了构建 Java EE 应用的事实标准。

特性

  • 核心技术：依赖关系注入，事件，资源，i18n，验证，数据绑定，类型转换，SpEL，AOP。
  • 测试： 模拟对象， TestContext 框架， Spring MVC Test， .WebTestClient
  • 数据访问：事务、DAO 支持、JDBC、ORM、编组 XML。
  • 春季 MVC和弹簧卷筒纸网络框架。
  • 集成：远程处理、JMS、JCA、JMX、电子邮件、任务、调度、缓存。
  • 语言：Kotlin，Groovy，动态语言。

优点

  • Spring是一个开源的免费的框架
  • Spring是一个轻量级、非入侵的框架
  • 控制反转(IOC), 面向切面编程(AOP)
  • 支持事务的处理， 对框架整合的支持

总结：Spring是一个轻量级的控制反转(IOC) 和面向且切面编程(AOP) 的框架

模块组成

IOC

动态的注入

在我们之前的代码中，需求可能会影响我们原来的代码， 我们需要根据用户的需求去修改原代码，如果程序代码十分庞大，修改一次的成本代价将会增加心智负担

我们使用Set接口实现动态的注入

之前是程序主动创建对象，控制权在自己手上

使用set注入后程序不再具有主动性，而是被动的创建对象, 我们可以再也不用去管理对象的创建了， 系统耦合性大大降低, 这也是IOC的原型

  • IOC的本质

ioc控制反转， 是一种设计思想，DI(依赖注入)是实现IoC的一种方法，也有人认为DI只是IoC的另一种说法，没有IoC的程序，我们使用面向对象编程，对象的创建与对象之间的依赖关系完全写死在程序中，对象的创建由程序自己控制，控制反转后将对象的创建转移给第三方，总结: 获得依赖对象的方式反转了

SpringIoC

IoC是Spring框架的核心内容使用对中方是完美的实现了IoC，可以使用XML配置，也可以使用注解, 新版本的Spring也可以零配置实现IoC。

Spring容器在初始化时先读取配置文件，根据配置文件或元数据创建与组织对象存入容器中， 程序使用再从IoC容器中取出需要的对象

采用XML配置文件方式配置Bean， Bean定义信息时和实现分离的， 而采用注解方式可以把两者合为一体，Bean的定义信息直接以注解方式定义在实现类中，从而达到了零配置的目的。

Bean作用域

Scope Description
singleton 单例模式, 永远只有一个对象
prototype 原型模式， 每次从容器中get的时候都会产生一个新的对象
request 将单个bean定义的范围扩大到单个HTTP请求的生命周期。也就是说，每个HTTP请求都有自己的bean实例，该实例是在单个bean定义的后面创建的。仅在感知web的Spring上下文中有效.ApplicationContext
session 将单个bean定义的范围扩大到HTTP的生命周期。只有在感知网络的Spring环境中才有效ApplicationContext
application 将单个bean定义作用域确定为。仅在web的Spring上下文中有效.ServletContext``ApplicationContext
websocket 将单个bean定义作用域确定为。只有在网络的Spring环境下才有效ApplicationContext

singleton

Bean 每发出请求时只会创建一个新的 Bean 实例

prototype

Bean 每发出请求时都会创新的 Bean 实例

其余的request、session、application这些只能在web中开发才能使用到

自动装配式Spring满足bean依赖一种方式。

装配的三种方式：

  1. 在xml文件中显示配置
  2. 在java中显示配置
  3. 隐式的自动装配bean

byname&&byType

byname的时候需要保证所有bean的id唯一，并且这个bean需要和自动注入的属性的set方法值一直

byType的时候需要保证bean的class唯一, 并且这个bean需要和注解的属性的类型唯一
