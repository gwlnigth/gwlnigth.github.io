生命周期

旧的组件生命周期图

挂载的生命周期

constroctor构造器

componentWillMount即将挂载

render 渲染页面

componentDidMount页面挂载完毕

componentWillUnmount 卸载页面

setState更新的生命周期

setState更新的生命周期顺序：shouldComponentUpdate -> componentUpdate -> render -> componentDidUpdate

shouldComponentUpdate 是否允许页面渲染

componentWillUpdate 页面即将渲染

render 渲染页面

componentDidUpdate页面挂载完毕

componentWillUnmount 卸载页面

forceUpdate更新的生命周期

forceUpdate更新的生命周期顺序：componentUpdate -> render -> componentDidUpdate

父组件的更新生命周期

父组件更新的生命周期顺序 componentWillReceiveProps -> componentUpdate -> render -> componentDidUpdate

componentWillReceiveProps当组件接收到新的Props参数的时候, 第一次的时候并不会执行

(新的React17.X)组件生命周期

  • 对于新的组件生命周期React17.x+后即将废弃componentWillMount、componentWillReceiveProps、componentWillUpdate。
    • 新版的需要加上前缀UNSAFE_比如UNSAFE_componentWillMount

新的组件生命周期图

对于旧版的新版的多了两个生命周期钩子

  • getDerivedStateFrimProps

  • getSnapshotBeforeUpdate

getDerivedStateFromProps

会导致state值永远是props

应用场景: state的值任何时候都是取决于props的时候可以使用这个生命周期钩子。

getSnapshotBeforeUpdate

类似于快照可以拿到更新之前的数据
